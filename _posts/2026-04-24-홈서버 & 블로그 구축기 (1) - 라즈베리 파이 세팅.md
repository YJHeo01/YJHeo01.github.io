# 라즈베리 파이 & Cloudflare 세팅

2026년 3월 어느 날, 단톡을 보다가 지인이 라즈베리파이 4B 4GB + 32GB SD카드 중고세트를 3만원에 처분한다는 글을 보고 바로 구매했다. 너무 저렴하지 않은가?


라즈베리파이를 충동 구매한 후, 이를 방치(...)하다가 홈서버를 구축하기로 결심했다.

---

## 1. 라즈베리파이 세팅 및 내부 접속

라즈베리파이를 세팅하는 방법은 크게 어렵지 않다. Raspberrypi Imager를 이용해서 SD카드를 구워준다. Raspberrypi Imager를 이용하면 OS 설치부터 계정 설정까지 쉽게 설정할 수 있기 때문이다.


나는 유선랜을 이용해서 라즈베리 파이와 공유기를 연결했다. 대부분 무선랜을 이용하는데, 유선랜을 사용한다고 해서 사용법이 달라지지는 않는다. 똑같이 터미널 창에서 `ssh <username>@<piname>.local`로 접속하면 된다. (.local은 mDNS 기반이라 별도 IP를 몰라도 접근이 가능하다.)

라즈베리파이를 세팅한 뒤 SSH를 통해 노트북과 연결했다. 이후 하단 명령어를 통해 간단히 웹 서버를 실행하여 내부 네트워크에서 접근 가능한지 확인했다.

```bash
python3 -m http.server 8000
```

브라우저에서 `http://라즈베리파이_IP:8000` 접속이 되면 정상적으로 작동하는 것이다.

---

## 2. 외부 접속 시도와 문제 발생

외부 접속을 위해 처음에는 공유기 포트포워딩을 고려했었지만, 접속이 되지 않는 문제가 발생했다.

확인 결과:

- 공유기 WAN IP: `192.168.x.x` (사설 IP)
- 실제 공인 IP: `curl ifconfig.me`로 확인됨

즉, 이중 NAT 환경이었음을 확인할 수 있었고, 이 구조에서는 단순 포트포워딩으로 외부 접속이 불가능하다. 따라서, Cloudeflare를 통해 터널링을 적용하기로 결정했다.

---

## 3. Cloudflare Tunnel 적용

따라서, 다음과 같은 구조를 나의 홈 서버에 적용하기로 하였다.

```text
브라우저 → Cloudflare → Tunnel → 라즈베리파이
```

터널링의 장점:

- 포트포워딩 불필요
- 공인 IP 필요 없음
- HTTPS 자동 적용

이를 적용하는 과정은 다음과 같다. (요금제는 Free 플랜)

1. 가비아에서 구매한 sk14cj.dev라는 도메인 주소를 Cloudeflare에 추가해준다.
2. 가비아에 있는 네임서버 값을 Cloudflare에서 할당한 값으로 수정한다.
3. 라즈베리파이에 cloudflared를 설치한다.
4. Cloudflare 대시보드를 통해 Tunnel을 한다.
5. 다음과 같은 키워드를 통해 재부팅할 때도 cloudflared를 자동으로 실행하게끔 설정한다.


```text
sudo systemctl restart cloudflared
```

---

## 4. nginx 도입

리버스 프록싱을 위해 nginx를 도입하여 다음과 같은 구조로 실행되도록 하였다.

```text
Cloudflare → nginx → 내부 서비스
```

nginx를 통해 다음과 같이 라우팅을 구성했다. 

```text
server {
    listen 80;
    server_name sk14cj.dev;

    # 메인 페이지
    location / {
        root /var/www/html;
        index index.html;
    }

    # 예비
    #location /api/ {
    #    proxy_pass http://localhost:5000/;
    #    proxy_set_header Host $host;
    #    proxy_set_header X-Real-IP $remote_addr;
    #}

}

```

---

## 5. HTTPS 구조 이해

https 등을 위해 nginx에서 443 포트 등도 열어야 하나 궁금했었는데, Cloudflare Tunnel을 사용할 경우 외부 HTTPS는 Cloudflare가 처리하기 때문에 그럴 필요가 없다고 한다.

```text
브라우저 (HTTPS)
      ↓
Cloudflare (HTTPS 종료)
      ↓
Tunnel (암호화된 연결)
      ↓
nginx (HTTP 80)
```

따라서 nginx에서는 80번 포트만 사용해도 충분하다.
